# 🤖 IMARA AI Integration Guide

Complete guide for integrating the AI disease detection service with your Express backend.

## 📋 Overview

```
Farmer uploads image
    ↓
Express Backend validates & saves image
    ↓
Calls Python FastAPI AI Service
    ↓
AI returns disease prediction
    ↓
Express saves with "pending_review" status
    ↓
Agronomist verifies/corrects prediction
    ↓
Farmer receives verified diagnosis
```

## 🚀 Quick Start

### Step 1: Set Up Python AI Service

```bash
# Navigate to AI service directory
cd IMARA-AI

# Create virtual environment
python -m venv venv

# Activate (Windows)
venv\Scripts\activate

# Activate (Linux/Mac)
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
```

### Step 2: Configure API Key

**Edit `IMARA-AI/.env`:**
```env
API_KEY=your-super-secure-api-key-2024
```

**⚠️ IMPORTANT:** Use the **SAME API KEY** in both services!

**Edit `IMARA-bn/.env`:**
```env
AI_SERVICE_URL=http://localhost:8000
AI_SERVICE_API_KEY=your-super-secure-api-key-2024
AI_SERVICE_ENABLED=true
```

### Step 3: Get Pre-trained Model

#### Option A: Download from Kaggle (Recommended)

1. **Visit**: https://www.kaggle.com/datasets/emmarex/plantdisease
2. **Download** the dataset and model
3. **Place model** at: `IMARA-AI/models/plantvillage_mobilenetv2.h5`

```bash
# Create models directory
cd IMARA-AI
mkdir models

# Place your downloaded model here:
# models/plantvillage_mobilenetv2.h5
```

#### Option B: Use Hugging Face Model

```bash
pip install huggingface-hub
```

```python
# download_hf_model.py
from huggingface_hub import hf_hub_download

model_path = hf_hub_download(
    repo_id="linkanjarad/mobilenet_v2_1.0_224_plant_disease",
    filename="model.h5",
    cache_dir="./models"
)
print(f"Model downloaded to: {model_path}")
```

#### Option C: Demo Mode (No Model Required)

The service works in demo mode without a model for testing!

### Step 4: Start AI Service

```bash
# In IMARA-AI directory with venv activated
python main.py
```

**Expected Output:**
```
✅ Model loaded successfully from ./models/plantvillage_mobilenetv2.h5
INFO:     Uvicorn running on http://0.0.0.0:8000
```

**Test the service:**
```bash

```

### Step 5: Start Express Backend

```bash
# In IMARA-bn directory
npm run dev
```

### Step 6: Install form-data Package

```bash
cd IMARA-bn
npm install form-data @types/form-data
```

## 🔑 Where to Get API Keys

### The API Key You Need

**There's NO external API to sign up for!** 

The API key is **generated by YOU** for security between your Express backend and Python AI service.

**How to create your API key:**

1. Generate a secure random string:
```bash
# On Windows PowerShell
[Convert]::ToBase64String((1..32|%{Get-Random -Max 256}))

# On Linux/Mac
openssl rand -base64 32

# Or use this simple string (change it!)
imara-ai-secure-key-$(date +%s)
```

2. **Copy the SAME key to BOTH places:**
   - `IMARA-AI/.env` → `API_KEY=your-generated-key`
   - `IMARA-bn/.env` → `AI_SERVICE_API_KEY=your-generated-key`

## 📊 API Endpoints

### Farmer Endpoints

**1. Upload Image for AI Detection**
```
POST /api/disease/detect
Headers: Authorization: Bearer {token}
Body: {
  imageUrl: string,
  cropId?: string
}
```

**2. View My Detections**
```
GET /api/disease?status=pending_review
```

### Agronomist Endpoints

**1. View Pending AI Detections**
```
GET /api/disease?verificationStatus=pending_review
```

**2. Verify AI Prediction**
```
PATCH /api/disease/{id}/verify
Body: {
  isCorrect: boolean,
  verificationNotes?: string,
  diseaseName?: string (if correcting),
  severity?: string (if correcting),
  treatment?: string,
  prevention?: string
}
```

## 🎯 Workflow Example

### Scenario: Farmer Detects Late Blight

1. **Farmer uploads** tomato leaf image
2. **AI predicts**: "Late Blight" (87% confidence)
3. **Status**: "pending_review" ⚠️
4. **Notification**: Agronomist receives alert
5. **Agronomist reviews** image and AI prediction
6. **Agronomist verifies** or corrects diagnosis
7. **Status updates**: "verified" ✅
8. **Farmer receives** verified diagnosis with treatment plan

## ⚠️ Important Notes

### 1. Always Require Agronomist Review

```typescript
// ✅ CORRECT - Show pending status
if (detection.verificationStatus === 'pending_review') {
  showWarning("⚠️ AI Prediction - Awaiting Expert Verification");
}

// ❌ WRONG - Don't let farmer act on unverified AI
if (detection.aiConfidence > 80) {
  allowFarmerToActImmediately(); // NO!
}
```

### 2. Collect Correction Data

Every time an agronomist corrects an AI prediction, save it for future model training:

```typescript
{
  aiPrediction: "Early Blight",
  aiConfidence: 75,
  correctedTo: "Late Blight",  // Agronomist correction
  verifiedBy: "agronomist-uuid"
}
```

### 3. Security

- ✅ Use HTTPS in production
- ✅ Rotate API keys regularly
- ✅ Validate image uploads (size, type)
- ✅ Rate limit AI endpoint
- ✅ Keep model files secure

## 🐛 Troubleshooting

### AI Service Not Reachable

**Error**: "AI Service not reachable"

**Solutions:**
1. Check if Python service is running: `curl http://localhost:8000/health`
2. Verify `AI_SERVICE_URL` in Express `.env`
3. Check firewall/ports (8000 should be open)
4. Review AI service logs

### Model Not Loading

**Error**: "Model file not found"

**Solutions:**
1. Check model path: `IMARA-AI/models/plantvillage_mobilenetv2.h5`
2. Download model (see Step 3 above)
3. For testing, service works in demo mode without model

### API Key Mismatch

**Error**: "Invalid API Key"

**Solutions:**
1. Ensure SAME key in both `.env` files
2. Check for typos or extra spaces
3. Restart both services after changing keys

### Low Confidence Predictions

**Issue**: AI confidence always < 50%

**Solutions:**
1. Check image quality (min 224x224px)
2. Ensure good lighting in photos
3. Crop image to show affected plant part
4. Consider fine-tuning model with local data
